# Sample run — committed, /home-free custody fixture

This directory is a **public fixture**: a real, deterministic VERDICT run committed
so a judge or reviewer can verify the custody chain offline with nothing but a
Python 3 interpreter. It is **not** customer evidence and **not** a clean bill of
health — it is the single-finding EVTX demo case that exercises the audit chain end
to end.

## What it is

A run over the public EVTX fixture `DE_1102_security_log_cleared.evtx` (a Windows
Security log carrying event ID 1102, "audit log cleared"). The run produces exactly
one CONFIRMED finding, `f-A-evtx-audit-log-cleared`
(MITRE ATT&CK **T1070.001**, Indicator Removal: Clear Windows Event Logs).

Files:

| File | What it is |
|------|------------|
| `audit.jsonl` | Hash-chained process + tool-call record (the Merkle leaf source) |
| `verdict.json` | Scoped Verdict and the single Finding |
| `run.manifest.json` | Signed manifest (ed25519, offline-integrity only) |
| `manifest_verify.json` | Offline verification result (`overall: true`) |

The verdict is `SUSPICIOUS` and the run is **not** customer-releasable: the signer
is `ed25519` (integrity, not identity) and expert sign-off is still required, so the
release gate stays at `EXPERT_REVIEW_DRAFT`. That is the honest state, not a defect.

## How it was regenerated

This fixture is reproducible. It was generated by the canonical one-shot path:

```bash
FINDEVIL_EVIDENCE_ROOT=/evidence scripts/verdict /evidence/DE_1102_security_log_cleared.evtx
```

The four files above were then copied verbatim out of `tmp/auto-runs/<case-id>/`.
No file in this directory is hand-edited — editing any of them breaks the hash chain
and `trace-finding` / `manifest_verify` will reject it.

The recorded provenance paths (e.g. `referenced_paths`, `verdict_artifact_path`,
`cryptographic_attestation.manifest_path`, the expert-miss `ledger_path`) are
relativized to basenames by `scripts/find_evil_auto.py::_release_path`, so this
public fixture carries **no absolute `/home/...` path**. Those fields are hashed
into the chain but never re-opened — `trace-finding` and `manifest_verify` resolve
by SHA-256 + `prev_hash`, not by opening the path — so relativizing the recorded
string keeps custody valid.

## Verify it yourself (offline, no MCP, no network)

```bash
# 1. Trace the verdict to its tool execution and Merkle leaves (exit 0 == OK)
scripts/trace-finding docs/release-evidence/sample-run

# 2. Confirm the offline verification result is overall: true
python3 -c "import json;print(json.load(open('docs/release-evidence/sample-run/manifest_verify.json'))['overall'])"

# 3. Confirm no local path leaked into the fixture
grep -rI '/home' docs/release-evidence/sample-run/   # prints nothing
```

`scripts/sample-run-trace-smoke.py` runs all three checks as a regression gate.
