# forensic-allowlist.txt — single source of truth for the OPTIONAL OS-level
# PreToolUse deny-hook (scripts/pretooluse-deny-hook.sh).
#
# This is a defense-in-depth layer BELOW VERDICT's in-process typed-MCP
# boundary, NOT a replacement for it. It defaults OFF; an operator opts in by
# wiring scripts/pretooluse-deny-hook.sh as a Claude Code Bash PreToolUse hook
# (see docs/sandbox/optional-os-hardening.md). When enabled, the hook hard-exits
# nonzero unless the FIRST invoked binary of a Bash command is on this list.
#
# The list is exactly the legitimate forensic binaries the typed MCP wrappers
# shell out to (grepped from services/mcp/src/tools/*). Adding a binary here is
# a reviewed, deliberate widening of the allowed OS surface — keep it minimal.
#
# Format: one binary basename per line. Blank lines and lines starting with '#'
# are ignored. Comparison is on the command's argv[0] basename, so both a bare
# name (`vol`) and an absolute path (`/usr/bin/vol`) match the same entry.
# Names ending in `.py`/`.exe` are matched as written (the wrappers invoke them
# by that exact basename).
